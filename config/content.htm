Onshape

Onshape is a computer-aided design (CAD) software system, delivered over the Internet via a Software as a Service (SAAS) model. It makes extensive use of cloud computing, with compute-intensive processing and rendering performed on Internet-based servers, and users are able to interact with the system via a web browser or the iOS and Android apps.

Onshape allows teams to collaborate on a single shared design, the same way multiple writers can work together editing a shared document via cloud services. It is primarily focused on mechanical CAD (MCAD) and is used for product and machinery design across many industries, including consumer electronics, mechanical machinery, medical devices, 3D printing, machine parts, and industrial equipment.

Onshape upgrades are released directly to the web interface, and the software does not require maintenance work from the user.

Founded in 2012, Onshape is based in Cambridge, Massachusetts (USA), with offices in Singapore and Pune, India. Its leadership team includes several engineers and executives who originated SolidWorks, a popular 3D CAD program that runs on Windows computers. Onshape’s co-founders include two former SolidWorks CEOs, Jon Hirschtick and John McEleney.

Onshape 是一種電腦輔助設計（ CAD ）軟件系統，通過軟件即服務（ SAAS ）模型在網路上提供。它廣泛使用雲端運算，在基於 Internet 的伺服器上執行計算密集型處理和彩現，用戶可以通過 Web瀏覽器 或 iOS 和 Android 應用程序與系統進行交互。

Onshape 允許團隊在單個共享設計上進行協作，就像多個編寫者可以通過雲端服務一起編輯共享文檔一樣。它主要專注於機械 CAD（ MCAD ），用於許多行業的產品和機械設計​​，包括消費電子，機械機器，醫療設備，3D 列印，機器零件和工業設備。

Onshape 升級直接發佈到 Web 界面，該軟件不需要用戶進行維護工作。

Onshape 成立於 2012 年，總部位於美國馬薩諸塞州劍橋市，在新加坡和印度浦那設有辦事處。其領導團隊包括幾位工程師和管理人員，他們發起了 SolidWorks，這是一種在 Windows 計算機上運行的流行 3D CAD 程序。Onshape 的聯合創始人包括兩名前SolidWorks 首席執行官，Jon Hirschtick 和 John McEleney。

    移除兩個零件間的所有自由度。

    在組合的兩個零件間建立轉動結合的限制條件。

    在組合的兩個零件間建立滑動結合的限制條件。

    Planar Mate (m) 平面結合

    在組合的兩個零件間建平面結合的限制條件。

    在組合的兩個零件間建立圓柱結合的限制條件。

    Independent Study

    Issue & Solution

    Design

    Q： Putting the mechanism to the V-rep is easy to interfere when simulating. And, the ball is easy to get stuck in one place in the process.

    A：Increase the slope or height of the track

    V-rep

    Q：Components cannot be separated

    A：There must be a gap between the part and the part, and put them together to be one part.

    Q：Turn on body is respondable, the components cannot operate on the normal track, it will flutter.

    A：Because the interval between the solid and the solid cannot be 0, there must be gap.

    Q：Turn on body is respondable for the track, the simulation will be very lag.

    A：Turn it to collide with the ball.

    Report

    Q："Pandoc" Path not found

    A：Need to set "path2" and start "path2" in "start.bat"

    Q："LaTeX" Modify name can't skip a line

    A：Use "for" loop to compile it

    Q：The teacher's name can't be displayed in pdf

    A：In "advisor_zh:", skip a line, add "-" and space to write

    Q：Can't transfer pdf in "Leo"

    A：Excute to specified path

    Q：Unbale to update catalog

    A：Modify the catalog name under "button Report pdf"

    Q：Images can't be displayed

    A：Path is wrong. Excute under "images"

    week12 - 6

    這週我們看了學長姐之前畫的鋼球運動系統，想說是否能找到靈感，但我們發現學長姐的專題是個人做一組機溝，他們做的方式是一個機構一個軌道就結束了，我們覺得有點太過於單一，所以我們決定繪製一組有多個機構組合而成的鋼球運動系統。

    week13 - 6

    這週於組員討論專體初步的的構想，從youtube上尋找一些靈感，我們找到一個不錯得機構(升降梯)，它是以馬達帶動偏心輪，使機構能作上下往復運動。
    零組件繪製完成遇到比較尷尬的問題是，它的作動不夠流暢，偏心輪，使機構上下往復運動配合有點問題，而且升降梯本身有斜度，必須精確的拿捏角度，這樣鋼球比較不會滾出去。

    week14 - 6

    這週工作是繪製新的機構(渦旋轉盤)，它是以馬達帶動渦旋轉盤做螺旋方式使鋼球上升，這機構遇到的問題點是它必須使用特徵掃掠方式去做出它的路徑，但是問題點是在它的螺旋線不太好畫，需要花很多時間去嘗試，而在做鋼球上升運動，我們發現鋼球無法從軌道掉下去，原因是因為，渦旋轉盤轉速過快，所以導致鋼球來不及進入軌道，所以我們加了一個擋板及加大出口，讓它有個緩衝時間，能使鋼球順利通過。

    week15 - 6

    這週繪製的機構(螺桿)，它是以馬達帶動螺桿作斜向上升運動，我們遇到的問題點是球無法進入螺桿的軌道，鋼球會被螺桿正在旋轉時被打出軌道，可能是角度的問題，但是我們沒去改軌道，因為更改軌道需要動到很多東西，所以我們想到另種方法，就是在螺桿上增加支架，使鋼球能順利上升。

    week16 - 6

    這週繪製的機構(飛輪)，它是以馬達帶動飛輪作旋轉運動使鋼球上升，這個機構跟螺桿機構是組裝在一起的，因為我們在軌道上做一個撥桿，使鋼球通過撥桿滾向不同的軌道，使鋼球能有規律的滾入到這兩個機構，今天所畫的機構就相對單純比較沒有問題發生，不太需要花很多時間在這個機構上。

    week17 - 6
